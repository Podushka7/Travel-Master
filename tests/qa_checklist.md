# Чек-лист тестирования подсистемы хранения данных 

Данный чек-лист разработан тестировщиком (Пашей ДеБаговым) для ручной и полуавтоматической проверки стабильности работы бинарной модели данных `TourDatabase`.

## 1. Проверки файловых операций (QDataStream / QFile)
- [ ] **Тест-01: Попытка чтения несуществующего файла.** При выборе в меню «Открыть...» бинарного файла, которого нет на диске, метод `db.load()` должен вернуть `false`, а интерфейс — вывести критическое предупреждение через `QMessageBox::critical`.
- [ ] **Тест-02: Чтение заблокированного файла или отсутствие прав доступа.** Проверка корректности обработки ситуации, когда файл `.tdb` открыт другой программой или защищен операционной системой от чтения.
- [ ] **Тест-03: Корректность сохранения счетчика ID при перезапуске.** Проверить, что после закрытия программы и повторного открытия бинарной базы переменная `m_nextId` восстанавливается из файла корректно, и новые создаваемые туры не вызывают коллизий и затирания старых ключей в `QMap`.

## 2. Валидация типов данных при бинарном считывании
- [ ] **Тест-04: Чтение поврежденного бинарного файла (Тест на избыточность).** Попытка скормить программе сторонний файл (например, переименованную картинку), сменив расширение на `.tdb`. Программа не должна падать с ошибкой сегментации (Segmentation fault); поток `QDataStream` должен безопасно завершить чтение, выдав ошибку валидации размера базы.
- [ ] **Тест-05: Валидация булевого флага визы.** Проверка того, что поле `visaRequired` при побайтовом чтении оператором `>>` корректно восстанавливает свое логическое значение (`true`/`false`) и правильно переносит состояние на графический чекбокс `checkVisa`.
- [ ] **Тест-06: Проверка лимитов частичного импорта N-записей.** Ввод в диалоговом окне `QInputDialog` числа $N$, превышающего реальное количество туров в архивном файле. Функция `qMin(N, allRecords.size())` должна корректно ограничить цикл и загрузить только доступные записи без выхода за границы вектора.
