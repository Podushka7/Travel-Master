# QA Checklist: Спецификация модульного тестирования (Unit Tests)

**Проект:** ИС «Travel-Master»  
**Фреймворк тестирования:** QTest (Qt Testlib)  
**Уровень тестирования:** Изолированное тестирование классов бизнес-логики (Model)  

---

## 1. Класс сущности `Tour`

### 1.1. Инициализация и базовые состояния
- [ ] **`testFirst`**: Проверка инициализации базового объекта (ID = -1, "Россия", 3000 руб).
- [ ] **`testParametricConstructor`**: Проверка инициализации объекта с пользовательским ID (в т.ч. отрицательным) и пустых строковых полей.

### 1.2. Сброс состояния (`resetToDefaults`)
- [ ] **`testResetToDefaultsType`**: Сброс вида отдыха на значение по умолчанию («пляжный»).
- [ ] **`testResetToDefaultsCountry`**: Сброс страны на «Россия».
- [ ] **`testResetToDefaultsPlace`**: Полная очистка поля места отдыха.
- [ ] **`testResetToDefaultsCost`**: Сброс цены путевки до базовой (3000 руб).
- [ ] **`testResetToDefaultsDuration`**: Сброс длительности тура на 2 дня.
- [ ] **`testResetToDefaultsVisa`**: Снятие флага визового режима.
- [ ] **`testResetToDefaultsMultiple`**: Комплексный сброс всех измененных полей одновременно.
- [ ] **`testResetToDefaultsAfterModify`**: Проверка стабильности многократного вызова сброса после серии модификаций.

### 1.3. Валидация данных
- [ ] **`testValidatorPlace`**: Тестирование регулярного выражения на кириллицу/латиницу, пробелы, дефисы и отсутствие цифр.
- [ ] **`testCreatePlaceValidatorUsingValidator`**: Проверка состояния валидатора (`QValidator::Acceptable` / `Invalid`) для корректных и некорректных строк ("Сочи" vs "Сочи2024").

### 1.4. Операторы сравнения
- [ ] **`testOperatorsLessEqual` / `testOperatorsGreaterEqual`**: Сравнение стоимости путевок (`<` и `>`).
- [ ] **`testOperatorsDifferentCost`**: Сравнение путевок с максимальным разбросом цен (крайние значения).
- [ ] **`testOperatorsSameCost`**: Инвертированное сравнение по длительности при равной стоимости (больше дней — выгоднее).
- [ ] **`testOperatorsEqualDifferentDuration`**: Сравнение объектов на равенство (`==`) строго по стране и месту, игнорируя цену и сроки.
- [ ] **`testOperatorsEdgeCases`**: Поведение операторов при нулевой стоимости и минимальной длительности.

## 2. Класс базы данных `TourDatabase`

### 2.1. Управление коллекцией (CRUD операции)
- [ ] **`testDatabaseAppend`**: Базовое добавление записи и проверка счетчика базы (`count`).
- [ ] **`testAppendMultipleRecords`**: Добавление серии записей подряд.
- [ ] **`testAppendCountIncrement`**: Строгий инкремент размера базы при каждой операции.
- [ ] **`testDatabaseRemove`**: Удаление записи и проверка смещения элементов.
- [ ] **`testDatabaseUpdate`**: Изменение полей существующей записи по ID и сохранение изменений в памяти.
- [ ] **`testDatabaseClear`**: Полное уничтожение данных и обнуление счетчиков.

### 2.2. Генерация идентификаторов и доступ к данным
- [ ] **`testAppendIdGeneration`**: Гарантия уникальности генерируемых ID.
- [ ] **`testAppendIdSequential`**: Проверка строгой последовательности присвоения ID (1, 2, 3...).
- [ ] **`testAppendRecordsRetrievable`**: Достоверность данных при вызове функции `record()` по ID.
- [ ] **`testRecordsEmpty` / `testRecordsSingleRecord` / `testRecordsMultiple`**: Валидация вектора записей для отображения в браузере в зависимости от наполненности базы.
- [ ] **`testRecordsOrder`**: Сохранение хронологического/логического порядка в возвращаемом векторе.
- [ ] **`testRecordsAfterRemove`**: Отсутствие пустых ссылок и «дыр» в векторе после удаления элемента в середине.

### 2.3. Интеграция флагов состояния
- [ ] **`testIsModifiedFlag`**: Проверка флага изменений при старте.
- [ ] **`testAppendModifiedFlag`**: Переключение флага в `true` при внесении данных.
- [ ] **`testIsModifiedFlag` (save cycle)**: Сброс флага в `false` после успешного сохранения на диск.

## 3. Бизнес-логика и вычисления

### 3.1. Агрегация данных (`totalCostByCountry`)
- [ ] **`testTotalCostByCountry`**: Базовый подсчет суммы стоимости путевок по одной стране.
- [ ] **`testTotalCostByCountryEmptyDb`**: Нулевая сумма при обращении к пустой базе.
- [ ] **`testTotalCostByCountryNonExistent`**: Игнорирование несуществующей страны.
- [ ] **`testTotalCostByCountrySingleRecord`**: Вычисление на основе единственного совпадения.
- [ ] **`testTotalCostByCountryMultipleCountries`**: Независимая агрегация при смешанных записях для разных стран.
- [ ] **`testTotalCostByCountryZeroCost`**: Корректная отработка алгоритма при путевках с нулевой стоимостью.
- [ ] **`testTotalCostByCountryLargeCosts`**: Защита от потери точности при сложении больших сумм (пограничные значения).

### 3.2. Манипуляция данными
- [ ] **`testSaleButton`**: Проверка алгоритма скидки (обновление стоимости путевки на 10% с округлением).
