# Устав проекта: Travel Master

## Команда проекта и распределение Scrum-ролей
Все участники команды являются разработчиками и пишут код, но для соблюдения фреймворка Scrum и успешного управления ЖЦ проекта за ними закреплены следующие роли:

*   **Богдан Ахлиманов** — Менеджер проекта / **Scrum Master**. Отвечает за координацию команды, ведение таск-трекера, устранение помех в работе и соблюдение регламента спринтов.
*   **Вася Питонов** — Ведущий разработчик / **Product Owner**. Отвечает за техническую стратегию, выбор архитектуры, управление бэклогом продукта и приоритеты функций (сериализация, хранилище `QMap`).
*   **Паша Багов** — Тестировщик / **Developer (QA)**. Отвечает за формирование тест-кейсов, проверку граничных условий UI, валидацию данных и аудит кода на соответствие принятому Guideline.
*   **Таня Сишник** — Разработчик / **Developer**. Отвечает за проектирование и верстку графического интерфейса (`mainwindow.ui`), связывание сигналов/слотов и точечное обновление виджетов.

## Цель проекта
Разработка удобного и безопасного локального приложения для автоматизации работы менеджеров туристических агентств.

## Границы проекта (Scope)
Программа реализует следующий строго ограниченный набор функций:
*   Создание карточек туров в фиксированный список мест (Россия, Абхазия, Турция, Египет).
*   Установка стоимости путевки и времени отдыха (в диапазоне дней от 2 до 31).
*   Автоматический учет и вывод отметки о необходимости визы для отдыха.
*   Фильтрация и динамическое отображение туров в браузере по выбранной стране.

## Регламент работы по методологии Scrum
1.  **Длина спринта:** На проекте приняты короткие спринты длительностью **1 неделя**.
2.  **Планирование спринта:** Проводится в начале каждой недели менеджером (Scrum-мастером) совместно с командой. Задачи переносятся из бэклога в таск-трекер.
3.  **Контроль задач:** Все текущие задачи фиксируются в таск-трекере. Карточка считается выполненной (Done) только после проверки тестировщиком (Пашей) и отправки кода в приватный репозиторий GitHub.
