# Бэклог проекта «Travel-Master»
**Связь с требованиями:** Данный документ реализует процесс управления задачами на основе [Спецификации требований (LAB3_REQUIREMENTS.md)](LAB3_REQUIREMENTS.md).

## 1. Бэклог проекта
Общий список требований (Epic/Features), выстроенный по приоритетам для реализации продукта:
1. **[Epic]** Разработка базовой архитектуры (Model-View) и структур данных (класс `Tour`).
2. **[Epic]** Создание главного окна (GUI) и связывание его с бизнес-логикой (Signals & Slots).
3. **[Feature]** Разработка подсистемы валидации ввода (регулярные выражения, QDoubleValidator).
4. **[Epic]** Разработка подсистемы сохранения/загрузки бинарных файлов (QDataStream).
5. **[Feature]** Интеграция всплывающих уведомлений (QMessageBox) об ошибках пользователя.

## 2. Бэклог спринта (Sprint Backlog)
**Цель Спринта №3:** Реализовать систему надежного локального хранения данных (сохранение и загрузка).

**Задачи, взятые в спринт:**
*   Разработать формат `.tdb` для бинарной сериализации данных.
*   Реализовать кнопку "Сохранить как..." и привязать к ней вызов `QFileDialog`.
*   Реализовать загрузку ранее сохраненного файла базы данных целиком.

### 2.1. Учет изменения требований 
В ходе текущего спринта произошла инициация риска: *Изменение требований со стороны заказчика*.
*   **Описание ситуации:** Изначально планировалась только загрузка файла базы данных целиком. Однако в процессе демонстрации промежуточных результатов стейкхолдер (Менеджер) запросил возможность частичного слияния баз (например, для переноса только свежих записей).
*   **Реакция на риск:** В бэклог текущего спринта была оперативно добавлена и реализована новая задача: 
    *   *«[Feature] Внедрить функцию чтения первых N-записей из выбранного файла `.tdb`»*.
*   **Итог:** Запрос заказчика удовлетворен, риск негативного влияния на проект купирован, задача успешно переведена в статус *Done*.
