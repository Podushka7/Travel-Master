## 2. Unit-тестирование (Автотесты)

Для автоматической проверки логики ядра и независимых вычислений (модуль `TourDatabase`) ведущим разработчиком (Васей) совместно с QA (Пашей) разработан банк тестов на базе встроенного фреймворка **Qt Test (`QT += testlib`)**.

### Код модуля автотестов (`database_test.cpp`)
```cpp
#include <QtTest>
#include "tourdatabase.h"
#include "tour.h"

class DatabaseTest : public QObject
{
    Q_OBJECT

private slots:
    void testEgyptSaleLogic(); // Тест автоматической скидки 10% для Египта
    void testBinarySerialization(); // Тест сохранения и загрузки QDataStream
};

void DatabaseTest::testEgyptSaleLogic()
{
    TourDatabase db;
    Tour t1(-1, "Отдых", "Египет", "Хургада", 50000, 10, true, 0);
    db.append(t1);

    // Вызываем функцию применения горящих туров
    db.applyEgyptSale(); 

    // База ассертов (Проверяем, что цена упала строго на 10% -> 45000)
    QCOMPARE(db.records().first().cost, 45000);
}

void DatabaseTest::testBinarySerialization()
{
    TourDatabase dbWrite;
    Tour t1(1, "Экскурсия", "Турция", "Кемер", 40000, 7, false, 0);
    dbWrite.append(t1);

    // Записываем в тестовый файл
    QVERIFY(dbWrite.save("test_dump.tdb") == true);

    TourDatabase dbRead;
    // Загружаем из тестового файла
    QVERIFY(dbRead.load("test_dump.tdb") == true);

    // Проверяем полноту и корректность восстановления структуры данных
    QCOMPARE(dbRead.count(), 1);
    QCOMPARE(dbRead.records().first().place, QString("Кемер"));
}

QTEST_APPLESS_MAIN(DatabaseTest)
#include "database_test.moc"
```
