# Журнал зарегистрированных дефектов (Bug Log) — Проект «Travel-Master»

Данный документ содержит реестр ошибок, обнаруженных в ходе Спринта №5 тестировщиком Пашей Баговым, и фиксирует статус их исправления разработчиками.

### Карточка Bug #1: «Логическая ошибка триггера визы для России»
*   **Тип дефекта:** UI / Логический
*   **Приоритет:** Высокий (High)
*   **Обнаружил:** Паша Багов (QA)
*   **Исполнитель:** Таня Сишник (Dev)
*   **Статус в багтрекере:** **[Closed / Done]**

**Описание проблемы:** 
Если менеджер на форме ввода сначала вручную устанавливал флажок «Нужна виза», а затем переключал текущую страну в комбобоксе на «Россия», флажок блокировался (`Disabled`), но отметка внутри него оставалась стоять (`Checked`). В результате путевка по России сохранялась в базу с некорректным требованием визы.

**Шаги для воспроизведения:**
1. Запустить приложение.
2. Поставить галочку в чекбокс «Нужна виза».
3. В выпадающем списке стран выбрать «Россия».
4. Нажать кнопку «Добавить тур».

**Решение проблемы:** 
Внутри файла `mainwindow.cpp` в слоте обработки смены страны `on_comboCountry_currentIndexChanged` добавлена принудительная очистка состояния чекбокса перед его блокировкой: `ui->checkVisa->setChecked(false);`. 
*   *Коммит с исправлением:* `fix: reset visa checkbox when Russia is selected` (Задача переведена в Done).

### Карточка Bug #1: «Коллизия ID при частичном импорте N-записей»
*   **Тип дефекта:** Архитектурный / База данных
*   **Приоритет:** Критический (Critical)
*   **Обнаружил:** Паша Багов (QA)
*   **Исполнитель:** Вася Питонов (Lead Dev)
*   **Статус в багтрекере:** **[Closed / Done]**

**Описание проблемы:** 
При вызове функции «Загрузить первые N записей» данные считывались из внешнего архива `.tdb` через поток `QDataStream` вместе со своими старыми идентификаторами `id`. Из-за этого при копировании объектов в текущую рабочую коллекцию `QMap` происходило наложение ключей, программа затирала существующие путевки менеджера или падала при попытке сгенерировать следующий ID.

**Шаги для воспроизведения:**
1. Создать и сохранить в файл `base1.tdb` два тура (они получат ID 1 и 2).
2. Создать чистую базу и сохранить в файл `base2.tdb` один тур (он получит ID 1).
3. Находясь в первой базе, вызвать импорт из файла `base2.tdb`.

**Решение проблемы:** 
В функцию побайтового чтения файла в `mainwindow.cpp` встроен принудительный сброс идентификатора считываемой структуры: `record.id = -1;` прямо перед добавлением в базу. Метод `db.append(record)` перехватывает это значение и автоматически генерирует для импортированного тура новый, уникальный в текущей сессии ключ на основе счетчика `m_nextId`.
*   *Коммит с исправлением:* `fix: resolve ID collisions in partial file import` (Задача переведена в Done).
