# Лабораторная работа №3. Требования
**Проект:** ИС «Travel-Master»

## 1. Формирование требований

### 1.1. Цель и задачи проекта
*   **Цель проекта:** Разработать автономное программное обеспечение (Desktop-клиент) для автоматизации рабочих процессов менеджера туристического агентства, обеспечивающее высокую скорость отклика и локальную безопасность данных.
*   **Задачи проекта:**
    1. Спроектировать графический интерфейс (GUI) для управления путевками.
    2. Разработать алгоритмы CRUD-операций для карточек туров.
    3. Реализовать систему локального бинарного сохранения данных (без использования внешних СУБД).
    4. Внедрить жесткую валидацию вводимых пользователем данных.

### 1.2. Функциональные требования (Functional)
*   **FR-1:** ИС «Travel-Master» должна обеспечивать возможность добавления новой записи о туре (страна, курорт, цена, длительность, наличие визы) в оперативно создаваемую базу данных.
*   **FR-2:** ИС «Travel-Master» должна отображать список всех загруженных туров в главном окне приложения в виде интерактивной таблицы/списка.
*   **FR-3:** ИС «Travel-Master» должна позволять выбирать тур из списка, редактировать его поля на лету и удалять записи (с блокировкой удаления последней записи).
*   **FR-4:** ИС «Travel-Master» должна поддерживать частичный импорт баз данных (возможность загрузки первых N-записей из выбранного внешнего файла с автосбросом ID для исключения коллизий).
*   **FR-5:** ИС «Travel-Master» должна по нажатию кнопки «Горящие туры» автоматически применять скидку 10% на все текущие туры в Египет.

### 1.3. Нефункциональные требования (Модель FURPS+)

#### Удобство использования (Usability)
*   **UR-1:** ИС «Travel-Master» должна обладать интуитивно понятным графическим интерфейсом, построенным на стандартных компонентах ОС (через Qt Widgets).
*   **UR-2:** ИС «Travel-Master» должна обеспечивать доступ ко всем основным функциям (CRUD) из одного главного окна без сложных вложенных меню.

#### Надежность (Reliability)
*   **RE-1:** ИС «Travel-Master» должна быть полностью автономной и сохранять работоспособность при отсутствии подключения к сети Интернет.
*   **RE-2:** ИС «Travel-Master» должна автоматически блокировать ввод некорректных данных (буквы в полях цены и дней, спецсимволы в названии курорта).
*   **RE-3:** ИС «Travel-Master» должна автоматически убирать и блокировать галочку «Нужна виза» при выборе страны «Россия».

#### Производительность (Performance)
*   **PE-1:** ИС «Travel-Master» должна мгновенно (в пределах 0.05 сек) обновлять список туров на экране благодаря удержанию активного контейнера данных в оперативной памяти.

#### Поддерживаемость и ограничения (Supportability +)
*   **SU-1:** ИС «Travel-Master» должна быть написана на языке C++ стандарта C++11 (согласно конфигурации проекта `CONFIG += c++11`).
*   **SU-2:** Графический интерфейс ИС «Travel-Master» должен быть реализован с использованием фреймворка Qt и системы сборки QMake.
*   **SU-3:** Система должна сохранять и считывать данные в кастомном бинарном формате `.tdb` через стандартный поток `QDataStream`.
