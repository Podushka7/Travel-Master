# Лабораторная работа №3. Требования
**Проект:** ИС «Travel-Master»

## 1. Формирование требований

### 1.1. Цель и задачи проекта
*   **Цель проекта:** Разработать автономное программное обеспечение (Desktop-клиент) для автоматизации рабочих процессов менеджера туристического агентства, обеспечивающее высокую скорость отклика и локальную безопасность данных.
*   **Задачи проекта:**
    1. Спроектировать графический интерфейс (GUI) для управления путевками.
    2. Разработать алгоритмы CRUD-операций для карточек туров.
    3. Реализовать систему локального бинарного сохранения данных (без использования внешних СУБД).
    4. Внедрить жесткую валидацию вводимых пользователем данных.

### 1.2. Функциональные требования:

*   **1)** ИС «Travel-Master» должна обеспечивать возможность добавления новой записи о туре (страна, курорт, цена, длительность, наличие визы) в оперативную память.
*   **2)** ИС «Travel-Master» должна отображать список всех загруженных туров в главном окне приложения.
*   **3)** ИС «Travel-Master» должна блокировать ввод некорректных данных (например, ввод строковых символов в поле «Цена»).
*   **4)** ИС «Travel-Master» должна осуществлять бинарную сериализацию данных и сохранять их в локальный файл формата `.tdb`.
*   **5)** ИС «Travel-Master» должна поддерживать частичный импорт баз данных (возможность загрузки первых N-записей из выбранного файла).

### 1.3. Нефункциональные требования:

**Удобство использования:**
*   **1)** ИС «Travel-Master» должна обладать интуитивно понятным графическим интерфейсом, построенным на стандартных компонентах ОС (через Qt Widgets).
*   **2)** ИС «Travel-Master» должна обеспечивать доступ ко всем основным функциям (CRUD) из одного главного окна без сложных вложенных меню.

**Надежность:**
*   **1)** ИС «Travel-Master» должна быть полностью автономной и сохранять работоспособность при отсутствии подключения к сети Интернет.

**Производительность:**
*   **1)** ИС «Travel-Master» должна мгновенно (без видимых задержек) обновлять список туров при добавлении или удалении записи благодаря использованию структур данных в оперативной памяти (QMap).

**Поддерживаемость и ограничения:**
*   **1)** ИС «Travel-Master» должна быть написана на языке C++ стандарта не ниже C++17.
*   **2)** Графический интерфейс ИС «Travel-Master» должен быть реализован исключительно с использованием фреймворка Qt 6.
